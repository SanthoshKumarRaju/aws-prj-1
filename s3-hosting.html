Hosting a Static Website on Amazon S3

Step-by-step guide to hosting a static website using Amazon S3

15 min read Beginner
1

Create an S3 Bucket

Log into AWS Management Console

Go to AWS Console and log in to your AWS account.

Navigate to S3 Service

In the AWS Console, type "S3" in the search bar or go to "Services" and select S3 under "Storage."

Create a New S3 Bucket

  • Click on the Create bucket button.
  • Bucket name: Choose a globally unique name, like dmi-staticwebpage.
  • Region: Select a region where you want the S3 bucket to be created (e.g., US East (N. Virginia)).
  • Leave the other settings as default unless you want to change any specific permissions or versioning options.
  • Click Create bucket to complete the process.
# No CLI commands needed - use AWS Console UI
2

Upload Files to the Bucket

Access Your Bucket

After creating the bucket, click on the bucket name (dmi-staticwebpage).

Upload Files

  • Click the Upload button.
  • Click on Add files to select the files from your local computer.
  • Select all the files you need to upload (including index.html and any other assets such as CSS, JS, images, etc.).
  • Once files are selected, click Upload.
3

Enable Static Website Hosting

Go to the Properties Tab

Once inside the bucket, click on the Properties tab.

Enable Static Website Hosting

  • Scroll down to the Static website hosting section.
  • Choose Enable.
  • For Index document, type index.html (the name of your homepage).
  • Optionally, for Error document, you can type error.html (if you have a custom error page).
  • Click Save changes.

Copy the Endpoint URL

After enabling static website hosting, AWS will provide you with a URL for accessing your site, like:

http://dmi-staticwebpage.s3-website-us-east-1.amazonaws.com/
4

Set Bucket Policy for Public Access

Go to the Permissions Tab

Inside the S3 bucket, click on the Permissions tab.

Edit the Bucket Policy

  • Scroll down to Bucket Policy and click Edit.
  • Add the following policy (replace the dmi-staticwebpage with your bucket name):
{
  "Version": "2012-10-17",
  "Statement": [
    {
      "Effect": "Allow",
      "Principal": "*",
      "Action": "s3:GetObject",
      "Resource": "arn:aws:s3:::dmi-staticwebpage/*"
    }
  ]
}

This policy allows anyone to view the files publicly on your static website.

Click Save changes.

5

Test the Static Website Access

Check the URL

After uploading your files, enabling static hosting, and setting the bucket policy, your website should be accessible at the S3 endpoint you copied earlier:

http://dmi-staticwebpage.s3-website-us-east-1.amazonaws.com/

Test Your Website

  • Open the URL in a web browser to check if your website is live.
  • If index.html loads correctly, your website is working.

Congratulations! Your static website is now hosted on Amazon S3.

Prerequisites

  • AWS Account
  • Static website files (HTML, CSS, JS)
  • Basic knowledge of AWS Console

Summary: S3 Static Website Hosting Advantages

What Makes S3 Hosting Special

Amazon S3 provides an exceptionally efficient way to host static websites with several distinct advantages:

Cost Effective

You only pay for storage and data transfer, with no ongoing server costs

High Performance

S3 delivers content with low latency and high throughput

Limitless Scaling

Automatically handles traffic spikes without any configuration

Reliable

Built on AWS's proven infrastructure with 99.9% availability

Ideal Use Cases

S3 static website hosting is perfect for:

  • Company brochures and marketing sites
  • Documentation and knowledge bases
  • Single Page Applications (SPAs) like React, Vue, or Angular
  • Static blogs built with generators like Jekyll or Hugo
  • Landing pages and campaign microsites
  • Portfolio websites for individuals and agencies

When to Consider Alternatives

While S3 is excellent for static content, consider these alternatives when you need:

Server-Side Processing

For dynamic content, consider EC2, Elastic Beanstalk, or Lambda

Advanced Routing

For complex URL rewriting, consider CloudFront Functions

WebSockets

For real-time applications, consider API Gateway with WebSockets

Enhancing Your S3 Website

Take your static website to the next level with these AWS services:

  1. CloudFront CDN: Distribute content globally for faster loading times
  2. Route 53: Use a custom domain name with DNS management
  3. ACM: Add free SSL/TLS certificates for HTTPS
  4. WAF: Protect your site from common web exploits
  5. Lambda@Edge: Run code at CDN locations for dynamic content

Cost Analysis Example

For a typical small business website (5GB storage, 50GB monthly transfer):

  • S3 Storage: $0.023 per GB = $0.12/month
  • Data Transfer: $0.09 per GB = $4.50/month
  • Requests: $0.0004 per 1000 requests = ~$0.20/month
  • Total: approximately $4.82/month

Compared to traditional hosting ($10-20/month), S3 offers significant savings for static sites.
