Deploy React Application on EC2 Instance

Step-by-step guide to deploy a React application on AWS EC2 with Nginx

20 min read Intermediate
1

Create an EC2 Instance

Launch an EC2 Instance

Go to AWS Console > EC2 > Instances > Launch Instance

Configuration

  • Name: react-app-server
  • AMI: Ubuntu Server 20.04 LTS
  • Instance type: t2.micro (free tier eligible)
  • Key pair: Create new or use existing key pair
  • Security group: Allow SSH (port 22), HTTP (port 80), and HTTPS (port 443)

Make sure to download and securely store your key pair file (.pem) for SSH access.

2

Install Node.js and npm

Connect to Your EC2 Instance

Use SSH to connect to your EC2 instance:

ssh -i "your-key.pem" ubuntu@your-instance-public-dns

Update System and Install Node.js

Run the following commands:

# Update package lists
sudo apt update

# Install Node.js and npm
sudo apt install -y nodejs npm

# Verify installation
node -v
npm -v

Explanation:

React applications require Node.js to run, and npm (Node Package Manager) to install dependencies. Node.js is the JavaScript runtime, and npm helps manage libraries and packages required by your application.

3

Install and Configure Nginx

Install Nginx

Nginx is a web server that will serve your React application:

# Install Nginx
sudo apt install -y nginx

# Start Nginx service
sudo systemctl start nginx

# Enable Nginx to start on boot
sudo systemctl enable nginx

# Check status
sudo systemctl status nginx

Explanation:

Nginx is a web server that will serve your React application. It will take care of handling HTTP requests and serving static files (like your React build).

4

Clone and Setup React Application

Clone from GitHub

Retrieve your React application from GitHub:

# Clone the repository
git clone https://github.com/pravinmishraaws/my-react-app.git

# Navigate to project directory
cd my-react-app

Modify the React App

Customize your application with your information:

# Edit the App.js file
nano src/App.js

Add your personalization:

// Example modification in App.js
<h2>Deployed by: <strong>Your Full Name</strong></h2>
<p>Date: <strong>DD/MM/YYYY</strong></p>
5

Build and Deploy React App

Install Dependencies and Build

Create production build of your React application:

# Install dependencies
npm install

# Create production build
npm run build

Deploy to Nginx Directory

Copy build files to Nginx web directory:

# Remove existing files
sudo rm -rf /var/www/html/*

# Copy new build files
sudo cp -r build/* /var/www/html/

# Set proper permissions
sudo chown -R www-data:www-data /var/www/html
sudo chmod -R 755 /var/www/html
6

Configure Nginx for React Routing

Update Nginx Configuration

Configure Nginx to handle React's client-side routing:

# Edit Nginx configuration
sudo nano /etc/nginx/sites-available/default

Replace the content with:

server {
    listen 80;
    server_name _;
    root /var/www/html;
    index index.html;
    
    location / {
        try_files $uri /index.html;
    }

    error_page 404 /index.html;
}

Restart Nginx

# Test configuration
sudo nginx -t

# Restart Nginx
sudo systemctl restart nginx
7

Access Your React Application

Find Your Public IP

Get your instance's public IP address:

# Get public IP
curl ifconfig.me

Access Your Application

Open your web browser and navigate to:

http://your-instance-public-ip

Congratulations! Your React application is now deployed on AWS EC2 with Nginx.

Summary: Key Takeaways for React Deployment on EC2

What We Accomplished

  • Successfully launched an EC2 instance with proper security groups
  • Installed Node.js runtime and npm package manager
  • Set up Nginx as a web server and reverse proxy
  • Cloned and customized a React application from GitHub
  • Built the React application for production
  • Configured Nginx to serve the React app and handle client-side routing
  • Accessed the deployed application via the EC2 instance's public IP

Why This Approach Works

Deploying a React application on EC2 with Nginx provides a cost-effective and flexible solution for hosting web applications. Unlike platform-as-a-service options, this approach gives you full control over the server environment while still leveraging AWS's reliable infrastructure.

Best Practices to Remember

Security

Always restrict SSH access to trusted IPs and use key-based authentication instead of passwords.

Performance

Enable Gzip compression in Nginx to reduce transfer size of your JavaScript bundles.

Deployment

Consider implementing a CI/CD pipeline to automate deployments when you push code changes.

Potential Challenges & Solutions

White Screen on Refresh

If your React app shows a white screen when refreshing specific routes, ensure your Nginx configuration includes the try_files $uri /index.html; directive to always fall back to index.html.

Environment Variables

For React apps needing environment variables, build the application on the server or use runtime configuration solutions like placing config files in the public folder.

SSL/HTTPS

Use Certbot with Let's Encrypt to add free SSL certificates to your Nginx configuration for secure HTTPS connections.

Where to Go From Here

Now that you have a working React deployment, consider these enhancements:

  • Set up a custom domain with Route 53 instead of using the IP address
  • Implement a process manager like PM2 to keep your Node.js app running continuously
  • Configure CloudFront CDN in front of your EC2 instance for improved global performance
  • Set up automated backups for your EC2 instance
  • Implement monitoring with Amazon CloudWatch to track server health
